“Nice” axis tick selection: 1/2/5 × 10^k steps, 4–6 ticks across a range.

Pure, testable tick math with no egui/wgpu deps. The renderer (app.rs) and the overlay (ui.rs) consume these to draw 3D tick marks plus numeric labels.

Functions§

fmt_tick
Format a tick value for the given axis and the visible span (span chooses RT units and, together with the tick count, the decimal precision so adjacent ticks render distinctly).
nice_step
Choose a “nice” step so that [lo,hi] is covered by ~4–6 ticks. Step ∈ {1,2,5}·10^k. Returns the step size (>0). target is the desired tick count.
ticks_for
Generate ticks across [lo,hi] (auto-orders lo/hi). First tick = ceil(lo/step)*step, stepping by step while <= hi (+epsilon). Each tick’s norm is computed by to_norm (pass the axis’s AxisTransform::normalize, or a window-aware normalizer).
